Student:

right now or later.
As a student, I want to be able to see my position in the queue and the estimated wait time so I can
manage my time.
As a student, I want to make sure the system will prevent others from cutting in line so that the queue is
fair
As a student, I want to book a slot with one or two taps so I can secure my meeting quickly.
As a student, I want to be promoted automatically and notified when a slot is open, so I don’t have to
refresh/re-request
As a student, I want to cancel my booking or leave the queue so the next student can be helped.

Fairness Logic Specification
Fairness Logic Specification
Overview
The system enforces fairness by limiting how often users can access meeting resources and by
protecting the time of users who have already waited in line. The goal is to ensure equal opportunity,
prevent abuse, and respect users’ time in a shared academic environment.
Implemented Fairness Rules
6
1. Meeting Usage Limit (Max Two Meetings per Week)
Each user is allowed a maximum of two meeting sessions per week.
If a user has already used both weekly meetings, they cannot join the queue again until the next week.
Why this is fair:
This rule prevents a small number of users from monopolizing limited meeting resources. In a
college/university setting, where many students may need access to advisors, tutors, or facilities, weekly
limits promote equitable access for everyone.
